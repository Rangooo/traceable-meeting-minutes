# 交叉验证渠道选择

根据待核查说法选择证据，不要套用单一来源排行榜。优先考虑六个维度：

1. **接近性**：是否直接记录、产生或观察到该事实
2. **权限性**：来源是否有资格确认该决策、数据或规则
3. **独立性**：多个来源是否真正独立，而非转载同一上游信息
4. **时效性**：资料是否覆盖会议讨论的时间点
5. **方法透明度**：指标定义、样本、计算方法和修订记录是否清楚
6. **利益冲突**：来源是否有动机选择性披露或夸大结论

## 按说法类型选择渠道

| 待核查说法 | 首选渠道 | 补充渠道 | 常见陷阱 |
|---|---|---|---|
| 会议中说了什么 | 原始录音、录像、逐字稿、聊天记录 | 多名参会人确认、会议笔记 | 把整理后的纪要当作逐字原话 |
| 是否作出决策 | 决策人确认、正式决议、签批或决策日志 | 原始会议记录、后续执行文件 | “讨论过”不等于“已决定” |
| 行动项及进度 | 负责人确认、工单或项目系统 | 邮件、日历、交付物、代码或文档记录 | 状态长期未更新；负责人只是被提及 |
| 内部经营数据 | 数据仓库、财务/业务系统、指标负责人 | 仪表盘、审计记录、抽样底表 | 仪表盘口径、刷新时间或权限范围不同 |
| 公司财务与经营披露 | 审计财报、监管申报、交易所文件、IR 材料 | 电话会、可靠财经数据库和媒体 | GAAP/non-GAAP、reported/organic、季度/年化混用 |
| 法律与监管要求 | 生效法条、官方公报、监管规则、判决原文 | 监管指南、专业法律数据库和律师意见 | 草案当现行规则；忽略司法辖区与生效日 |
| 技术能力或产品行为 | 可复现测试、源代码、官方技术文档、版本记录 | 标准、基准测试、独立技术分析 | 文档版本与实际部署版本不一致 |
| 科学或医学主张 | 同行评审研究、系统综述、权威指南、原始数据 | 专业机构、预注册及后续研究 | 单篇论文、代理指标或相关性被当成因果 |
| 市场规模与行业趋势 | 方法透明的统计数据、行业数据库、原始调研 | 多家独立研究、公司披露、专业媒体 | 不同定义和样本被直接比较 |
| 客户体验与舆情 | 原始访谈、工单、调查、评论和社区讨论 | 客服数据、留存/转化行为指标 | 社交平台样本偏差；声量不等于占比 |
| 新闻与公共事件 | 当事方原始声明、现场记录、政府或机构资料 | 多家独立可靠媒体、事实核查机构 | 多家媒体引用同一匿名消息源 |

## 组合证据

- 对高影响结论，尽量组合“一手记录 + 有权限的确认 + 独立证据”。
- 同源转载只计算为一个来源；明确共同的上游来源。
- 证据冲突时比较时间、定义、覆盖范围和修订版本，不按来源数量投票。
- 无法访问最合适渠道时，标注限制，并说明下一步应向谁、在哪个系统或用什么资料核查。
- 参与者确认属于外部沟通行为；除非用户明确授权，不代替用户发消息或联系他人。
